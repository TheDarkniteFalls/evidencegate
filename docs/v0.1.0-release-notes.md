# EvidenceGate v0.1.0 Release Notes

EvidenceGate v0.1.0 is the first tagged source release of its revision-bound
receipt contract for human-reviewed AI-assisted Git changes.

## Included

- strict v1 receipt JSON Schema and relational validation;
- read-only comparison with a clean local Git checkout;
- claim-to-check, exact-head, complete-diff, allowed-path, protected-path, human
  review, and public-safety relationships;
- deterministic Markdown rendering that neutralizes supplied structure;
- portable positive and negative conformance fixtures with independent Python
  and Node consumers; and
- `evidencegate_cli_result_v1` JSON results with stable finding codes for
  `validate` and `verify` integrations.

Install the pinned source release with:

```sh
python -m pip install \
  "evidencegate @ git+https://github.com/TheDarkniteFalls/evidencegate.git@v0.1.0"
evidencegate --self-test
```

Version 0.1.0 is an unsigned source tag and GitHub release. It is not uploaded
to PyPI.

## Important Boundaries

EvidenceGate records and checks supplied evidence. It does not rerun commands
from arbitrary receipts, authenticate reviewer identities, prove that a change
is correct or secure, or authorize merge or publication. The included in-toto
profile is unsigned and deliberately reports `UNAUTHENTICATED`; the reviewer
study is a protocol, not a measured external result.

The unsigned tag and GitHub release identify the clean revision that passed the
release checklist. No package-index upload is part of v0.1.0.
